# 项目需求文档

> 版本: 1.0 | 更新日期: 2026-07-13

---

## 项目背景

开发一款适用于机械臂末端的空耦超声检测系统。在保证系统精简的前提下，实现超声探头的激励与同步采集、数据上传、设备间通信等功能。

## 应用场景

- 工业机械臂末端安装
- 非接触式(空耦)超声检测
- 需要高刷新率实时波形显示
- 设备需要长时间稳定运行

## 硬件平台

| 项目 | 当前平台 | 规划升级平台 |
|------|---------|-------------|
| MCU | STM32F103RC (Cortex-M3, 72MHz) | STM32F407VG / H743 |
| RAM | 48KB | 192KB+ |
| Flash | 256KB | 1MB+ |
| 采集 | ADC1, 12-bit, ~1MSPS | 三重交错ADC, 7.2MSPS |
| 通信 | USART1, 921600 baud | USB CDC / 10.5Mbps UART |
| 显示 | SSD1306 OLED 128×64 I2C | 同规格 |
| 激励输出 | DAC1, 12-bit, PA4 | 同规格 |

## 功能需求

### FR-1: 超声激励信号输出
- 频率: **100 kHz** (可配置)
- 周期数: **5 个** (可配置)
- 周期: **10 ms** (可配置)
- 波形: 方波 (预留正弦波表支持)
- 输出方式: DAC + DMA + 定时器触发

### FR-2: 同步数据采集
- 采样率: **≥ 814 kS/s** (实测 ~1 MS/s @ 12MHz ADC时钟 + 1.5周期采样)
- 每次采集点数: **1000 点** (可配置)
- 采集触发: 与激励同步 (同一TIM2 ISR)
- 缓冲方式: 双缓冲 (ping-pong)

### FR-3: 端侧波形算法处理
- 最大幅值提取
- 渡越时间 (TOF) 计算 — 阈值交叉法
- RMS (均方根值) 计算
- 信号能量计算
- 检测阈值可配置

### FR-4: 自定义串口协议通信
- 帧格式: 帧头 + FrameID + CMD + PayloadLen + Payload + CRC16-Modbus
- 支持上下行双向通信
- 自适应帧率: 全帧(N帧1次) + 特征帧(每帧)
- ACK/NACK 应答机制
- 上位机指令: 启停/参数配置/状态查询

### FR-5: 上位机显示
- 刷新率: **38 FPS** 目标
- 波形实时显示
- 特征数据图表
- 参数配置面板

### FR-6: 系统可靠性
- IWDG 独立看门狗 (~1秒超时)
- 多任务心跳监控
- 故障异常诊断信息输出
- UART/ADC/DMA 错误计数与恢复
- 任务栈水位监控

### FR-7: OLED 状态显示
- 页0: 实时指标 (ADC速率 / UART速率 / CPU使用率)
- 页1: 系统健康 (各任务栈使用 / 堆使用 / 运行时间 / 错误计数)
- 页2: 信号特征 (幅值 / TOF / RMS / 能量)
- 每3秒自动翻页

## 非功能需求

### NFR-1: 实时性
- TIM2 ISR 抖动 < 1μs
- ADC 完成中断到 ProcessTask 唤醒延迟 < 500μs
- 控制命令响应延迟 < 10ms (一帧周期内)

### NFR-2: 可靠性
- 连续运行 > 72小时无死机
- 看门狗复位后 1秒内恢复工作
- 栈使用率 < 80% (留有安全余量)

### NFR-3: 可移植性
- 应用层代码不依赖特定 HAL
- 换芯片平台仅需替换 hal_port 实现文件

### NFR-4: 可维护性
- 所有模块有 Doxygen 标准注释
- 关键设计决策有文档记录
- 架构图嵌入代码注释
