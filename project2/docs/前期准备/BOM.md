## 一、MCU选型分析

### 方案对比

| 维度 | 方案A: STM32G431 | 方案B: STM32F411（已有） | 方案C: G431套件 |
|------|-----------------|------------------------|----------------|
| 内核 | M4F @ 170MHz | M4F @ 100MHz | M4F @ 170MHz |
| 高级定时器 | TIM1 + **HRTIM** | TIM1（互补PWM+死区） | 同方案A |
| 内置PGA运放 | ✅（简化电流采样） | ❌（需外部处理） | 同方案A |
| 内置比较器 | ✅（硬件过流保护） | ❌（软件ADC轮询） | 同方案A |
| RAM | 32KB（够用但紧） | **128KB（宽裕）** | 同方案A |
| ST电机SDK | ✅ MCSDK | ❌ 需SimpleFOC或手写 | 同方案A |
| 6步换相 | ✅ | ✅ **完全够用** | ✅ |
| 基础FOC | ✅ 顺畅 | ✅ 可行，多些折腾 | ✅ |
| 面试叙事 | "选型理由：电机专用芯片" | "在通用MCU上实现FOC，理解了专用芯片的价值" | 同方案A |
| 预算 | ~150元 | 0元（已有） | ~200-300元 |

### 方案A vs 方案B 深度对比

**F411完全可以做这个项目。** 具体分析：

1. **PWM**: F411的TIM1有6路互补PWM+硬件死区插入，做6步换相和SVPWM都足够。G431的HRTIM是锦上添花（PWM分辨率从~13bit提到~16bit），对项目级别无实质影响。

2. **FOC电流环**: 都需要ADC做相电流采样（**INA226是I2C接口，速度太慢只能用于SOC，不能用于FOC电流环**）。F411的ADC 2.4MSPS完全够。G431的优势是内置运放可以省外部电路，但如果你用MS8313驱动板（板上通常有电流检测引脚），这个优势也被绕过了。

3. **过流保护**: G431的硬件比较器是μs级响应。F411只能ADC轮询（ms级），但MS8313驱动芯片自带硬件过流保护——所以实际保护靠驱动芯片，不靠MCU。

4. **开发体验**: G431有ST MCSDK可一键生成FOC代码框架。F411只能用SimpleFOC（偏Arduino生态，STM32上需移植）或纯手写。F411的开发路径更颠簸但学到的更多。

5. **面试叙事**: 两个方向都成立——
   - G431: "我选了ST的电机控制专用芯片，因为有HRTIM和内置PGA"
   - F411: "我在没有专用外设的通用MCU上实现了FOC，这让我理解了为什么G4系列要加HRTIM和比较器"

### 建议

- **预算充裕** → 买G431套件（方案C），开箱即用最省时间
- **预算紧张** → 用手里F411（方案B），多花时间但能学到更多底层
- **折中** → 单独买NUCLEO-G431RB + MS8313模块 + 配件自己焊

## 四、限流电源说明

### 为什么要限流电源

写BLDC驱动代码时，最常见的bug是**上下桥直通短路**：

```
换相逻辑写错 → 同相上下桥MOS管同时导通
              → 12V经两个MOS管直接短路到地
              → 电流瞬间飙升到几十A
              → 无保护 → MOS管冒烟烧毁 → 驱动板报废
```

**有限流电源的保护链：**

```
代码bug → 电流冲到设定上限(如2A) → 电源自动降低输出电压
        → 驱动板安全 → 电机抖动/不转 → 你知道有bug → 修代码 → 重试
```

### 使用方式

调试时先设电压=12V、电流上限=1-2A。电机正常运转电流通常在0.3-1A范围。如果电机不转且电源显示恒流灯亮 → 有短路bug。

### 替代方案

如果预算实在紧张，退而求其次：
- 在电源和驱动板之间串一个**2A保险丝座**（约5元）+ 备几个2A保险管
- 短路 → 烧保险丝 → 换一颗 → 比烧整个驱动板便宜
- 缺点：反应慢（保险丝熔断需ms级），不如限流电源快

## 五、电源架构设计

### 5.1 系统电压需求

```
MS8313驱动板: 8~30V DC ──→ 接12V电源或11.1V电池
MCU开发板:    5V USB    ──→ 电脑USB供电（调试期间），或7-12V VIN
INA226:       3.3V      ──→ 从MCU板3.3V取电
OLED:         3.3V      ──→ 从MCU板3.3V取电
BLDC电机:     三相交流   ──→ 由MS8313驱动板输出
```

### 5.2 两种工作模式的供电拓扑

#### 模式A：纯电机调试（不测SOC）

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   12V电源适配器 ──→ MS8313驱动板 VIN(+) ──→ BLDC电机    │
│        │                    │                           │
│        │                    └── GND ──────────────┐      │
│        │                                          │      │
│        └── (如果12V适配器也供电MCU)                │      │
│             → MCU VIN (7-12V)                     │      │
│             → 或：MCU另由USB供电，仅GND共地 ──────┘      │
│                                                         │
│   MCU开发板 ──USB──→ 电脑（烧录+串口监控）              │
│                                                         │
│   注意：MCU GND 和 驱动板 GND 必须连接！（共地）         │
└─────────────────────────────────────────────────────────┘
```

#### 模式B：SOC放电测试（电池监测）

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│   3S航模电池(11.1V)                                          │
│       │                                                      │
│       ├── 正极(XT60红) ──→ INA226 VIN+ ──→ INA226 VOUT+     │
│       │                          │                           │
│       │                     I2C→MCU (测电压+电流)            │
│       │                          │                           │
│       │                          └──→ MS8313驱动板 VIN(+)    │
│       │                                    │                  │
│       │                                    └──→ BLDC电机     │
│       │                                                      │
│       └── 负极(XT60黑) ──→ INA226 GND ──→ MS8313 GND        │
│                                          ──→ MCU GND (共地!) │
│                                                              │
│   MCU开发板 ──USB──→ 电脑（串口看SOC/电流/转速数据）        │
│                                                              │
│   原理：电池→传感器→负载→电机，INA226串联在中间测一切       │
└──────────────────────────────────────────────────────────────┘
```

### 5.3 两种模式对照

| | 模式A: 纯电机调试 | 模式B: SOC放电测试 |
|---|---|---|
| 电机供电 | 12V电源适配器 | 3S航模电池 |
| MCU供电 | USB | USB |
| INA226 | 不接/旁路 | **串联在电池和负载之间** |
| 测什么 | PWM/转速/电流波形 | SOC下降曲线/电池放电特性 |
| 什么时候用 | Phase 1（电机控制开发） | Phase 2-3（SOC算法开发验证） |

### 5.4 INA226接线要点

```
INA226模块引脚:
  VIN+  ← 电池正极
  VIN-  → 接驱动板VIN(+)  [注意：电流流过采样电阻]
  GND   ← 电池负极 + 驱动板GND + MCU板GND（共地）
  SCL   → MCU I2C_SCL (PB6 或 PB8)
  SDA   → MCU I2C_SDA (PB7 或 PB9)
  VCC   → MCU 3.3V

⚠️ 关键：INA226是串联在电源回路里的，不是并联！
   VIN+ 和 VIN- 之间内部有一个0.1Ω采样电阻
   电流流过这个电阻 → 产生微小的压降 → INA226放大并数字化 → I2C读出
```

## 六、完整接线规划

### 6.1 引脚分配

```
                    MCU (NUCLEO-G431RB 或 F411)
                    ═══════════════════════════
                    
MS8313驱动板                            INA226 + OLED (共用I2C1)
────────────────                        ──────────────────────
PA8  → INHA (A相上桥PWM)               PB6 → SCL
PA9  → INHB (B相上桥PWM)               PB7 → SDA
PA10 → INHC (C相上桥PWM)
PB13 → INLA (A相下桥PWM)               串口上报
PB14 → INLB (B相下桥PWM)               ────────
PB15 → INLC (C相下桥PWM)               PA2 → USB-TTL RX
                                        PA3 → USB-TTL TX
ADC电流采样(可选，FOC电流环用)
──────────────────────
PA0  → 驱动板电流检测输出引脚           调试/按键
（具体引脚需看MS8313模块原理图）         ────────
                                        PC13 → 板载LED（打点用）
                                        PB0  → 按键（翻页OLED）
```

### 6.2 I2C设备地址

```
I2C1总线上挂两个设备：
  INA226:  0x40 (A0=GND, A1=GND)
  SSD1306:  0x3C (固定)
  地址不冲突，可共用同一组SCL/SDA
```

## 八、采购建议

1. **MCU**: 先算账。G431套件如果250元以内且包含电机——性价比之王。如果超300，用F411顶上去。
2. **驱动板**: MS8313关键字直接搜淘宝，买带接线端子的模块（不要买裸片），确认引出电流检测引脚。
3. **电机**: 2204/2205 KV1400，买带香蕉头/硅胶线的，方便接驱动板。顺便问卖家要电机参数（极对数、相电阻、KV值）。
4. **INA226**: 买成品模块，板上已有0.1Ω采样电阻+接线端子，不用自己焊。
5. **电池**: 3S 2200mAh带XT60插头。顺便买一个XT60母头转接线用于面包板连接。
6. **可调电源**: 闲鱼淘二手（许多学生毕业出），30元以内能收到。再不济搜"DC-DC可调降压模块"自己加个12V适配器凑合。
