## 1. 核心设计原则

### 1.1 数据流优先 (Data Flow First)

在设计任何新功能前，先在纸上画出数据流路径。明确:
- 数据从哪里产生 (ISR / Task / Peripheral)
- 经过哪些缓冲 (Double Buffer / Queue)
- 谁消费 (Task / UART TX)
- 可能拥塞的点 (Queue深度 / 带宽瓶颈)

### 1.2 失效安全 (Fail-Safe)

- 默认状态是安全的 (上电=不转, 需命令启动)
- 看门狗是最后防线
- 任一组件失效不能导致整系统崩溃
- **新增**: 电机过流 → 立即停PWM输出，进入 FAULT 状态
- **新增**: 电池欠压 → 禁止启动电机，OLED闪烁告警
- 错误累积后自动降级 (进入 FAULT 状态)

### 1.3 可观测性 (Observability)

- 关键指标必须可视化 (OLED / 串口可查询)
- 异常必须有诊断输出
- 运行时性能必须是可测量的 (打点+示波器/逻辑分析仪)
- **新增**: 电机运行时OLED显示实时转速/电流波形（文本模拟）

### 1.4 平台无关性 (Platform Independence)

- 应用逻辑不直接依赖 HAL API
- 通过 `hal_port.h` 接口间接访问硬件
- 换芯片 = 换一个 `hal_port_xxx.c` 文件

## 2. 模块设计模板

每个新模块必须包含以下注释块:

```c
/**
 * @file    module_name.c
 * @brief   模块一句话描述
 * @ingroup ModuleGroup
 *
 * ## 职责 (Responsibility)
 * 本模块负责: ...
 * 本模块不负责: ...
 *
 * ## 数据流 (Data Flow)
 * 输入:  [来源] → 本模块
 * 输出:  本模块 → [去向]
 *
 * ## 并发安全性 (Concurrency)
 * - 哪些函数从 ISR 调用: ...
 * - 哪些函数从任务调用: ...
 * - 共享资源保护方式: ...
 *
 * ## 使用示例 (Usage)
 * ```
 * Module_Init();
 * Module_Start(param);
 * ```
 *
 * ## 设计决策 (Design Decisions)
 * - 为什么用 A 而不用 B: ...
 * - 为什么选这个阈值/周期: ...
 */
```

## 3. 中断服务程序 (ISR) 设计规范

### 3.1 ISR 编码铁律

1. **极短**: ISR 执行时间 < 5μs
2. **无阻塞**: 绝不调用阻塞 API (如 osSemaphoreAcquire)
3. **可调用**: osSemaphoreRelease / osMessageQueuePut (fromISR 变体) / TaskNotifyGive
4. **有浮点**: Cortex-M4F 有 FPU，但ISR内仍需节制——仅简单赋值
5. **有清理**: 必须清除中断标志，防止重入

### 3.2 ISR → Task 延迟处理模式

```c
// ISR 中只做最小工作
void ISR_Handler(void) {
    ClearFlag();                    // 清标志
    osSemaphoreRelease(semHandle);  // 通知任务
    // 不做实际处理!
}

// Task 中做实际工作
void Task_Handler(void) {
    for(;;) {
        osSemaphoreAcquire(semHandle, osWaitForever);
        // 这里做实际处理
    }
}
```

## 4. DMA 设计规范

### 4.1 DMA 优先级分配

| DMA 通道 | 外设 | 方向 | 优先级 | 理由 |
|----------|------|------|--------|------|
| DMA1_CH1 | ADC1 | P→M | HIGH | 电流采样不能断 |
| DMA1_CH2 | USART2 TX | M→P | LOW | 非实时，大数据量可打断 |
| DMA1_CH3 | USART2 RX | P→M | MEDIUM | 命令数据小但需及时响应 |

### 4.2 DMA 模式选择

- **Normal 模式**: 简单、可控。完成后手动重启。用于:
  - ADC 电流采样 (配合双缓冲，完成中断中切换)
  - USART TX (变长帧)
- **Circular 模式**: 连续不间断。用于:
  - (本项目暂不使用，避免 HAL 状态机复杂化)

## 5. 电机控制安全规范 (新增)

### 5.1 启动安全

- 软启动：占空比从0%线性增加到目标值，速率 ~5%/100ms
- 启动前检查：电池电压 > 9.0V，DRV8313 无FAULT信号
- 堵转检测：启动2秒后转速 < 100RPM → 判定堵转，停机

### 5.2 运行安全

- 过流保护：软件阈值 = 5A (INA226读数)，触发 → 立即停PWM
- 过温保护：MCU温度 > 80°C → 降功率运行
- 通信丢失保护：连续500ms未收到心跳 → 进入安全模式(停电机)

### 5.3 死区时间

- 上下桥PWM插入死区 500ns (HRTIM自动插入)
- 防止上下桥直通短路

### 5.4 紧急停止

```c
// 紧急停止函数 — 可在ISR中调用
void Motor_EmergencyStop(void) {
    // 1. 立即关闭所有PWM输出 (输出低电平)
    HAL_TIM_PWM_Stop(&htim1, TIM_CHANNEL_1);
    HAL_TIM_PWM_Stop(&htim1, TIM_CHANNEL_2);
    HAL_TIM_PWM_Stop(&htim1, TIM_CHANNEL_3);
    // 2. 设置状态为FAULT
    g_motorState = MOTOR_FAULT;
    // 3. 记录故障原因
    Error_Report(ERR_MOTOR_OVERCURRENT);
}
```

## 6. 内存管理规范

### 6.1 内存预算 (32KB SRAM)

| 用途 | 大小 | 说明 |
|------|------|------|
| FreeRTOS 堆 | 12 KB | 任务栈+队列+信号量 |
| FOC运行时数据 | 2 KB | Clarke/Park变换中间变量、PI积分项 |
| ADC 双缓冲 | 1 KB | 电流采样 256×2×2 bytes |
| 串口缓冲区 | 1 KB | txBuf |
| OLED 显存 | 1 KB | 128×64/8 |
| 系统栈 | 1 KB | startup |
| 全局/静态数据 | ~2 KB | .bss + .data |
| **已用** | **~20 KB** | |
| **剩余** | **~12 KB** | 安全余量充足 |

## 7. 命名规范

### 7.1 文件命名

```
User/
  MotorControl/    foc.c/h             BLDC磁场定向控制
                   bldc_commutation.c/h 6步换相基础驱动
  BatteryMonitor/  ina226.c/h           INA226 I2C驱动
                   soc.c/h              SOC估算算法
  Display/         oled_display.c/h     OLED显示
  Communication/   uart_report.c/h      串口数据上报
  Common/          error_handler.c/h    错误处理
                   hal_port.h           HAL抽象接口
```

### 7.2 函数命名

- 公开API: `Module_Action()` 如 `FOC_Init()`, `SOC_GetPercentage()`
- 静态函数: `ActionSubject()` 如 `Clarke_Transform()`, `Coulomb_Count()`
- ISR回调: `HAL_Peripheral_CallbackType()` (HAL 规范)
- 模块前缀: `FOC_`, `BLDC_`, `INA226_`, `SOC_`, `OLED_`, `UART_`, `Error_`

### 7.3 变量命名

- 全局: `g_` 前缀 如 `g_motorState`, `g_socPercent`
- 静态文件级: `s_` 前缀 如 `s_piIntegral`
- HAL 句柄: `h` + 外设名 如 `htim1`, `hi2c1`, `huart2`
- FreeRTOS 对象: `Handle` 后缀 如 `CurrentLoopSemHandle`

## 8. 错误处理规范

### 8.1 错误分级

| 级别 | 响应 | 示例 |
|------|------|------|
| FATAL | 记录 + 立即停电机 + 等IWDG复位 | 过流、堵转、HardFault |
| ERROR | 记录 + 外设复位重试 | INA226 I2C超时、UART帧错误 |
| WARNING | 仅计数 | SOC估算抖动超阈值 |

### 8.2 返回值检查

所有 HAL API 调用必须检查返回值:
```c
if (HAL_I2C_Master_Transmit(&hi2c1, addr, buf, len, timeout) != HAL_OK) {
    Error_Report(ERR_I2C_TIMEOUT);
    return;
}
```

## 9. 测试与验证规范

### 9.1 每个Phase结束后的验证清单

- [ ] 编译通过 (0 Error, 0 Warning)
- [ ] 电机正常运行 (Phase 1后)
- [ ] OLED 所有页面正常显示
- [ ] 串口数据格式正确，Python脚本能解析
- [ ] 看门狗: 手动制造死循环 → 系统复位 + 电机停止
- [ ] 过流保护: 模拟短路 → 电机立即停止
- [ ] 栈高水位: 运行30分钟后所有任务 > 30% 余量

### 9.2 性能测量方法

1. 在空闲 GPIO (如 PC13 LED) 打点
2. 逻辑分析仪 / 示波器测量脉冲宽度 = 代码段执行时间
3. 关键测量点:
   - FOC电流环单次迭代时间 (target < 25μs @ 20kHz for 50% CPU)
   - ISR入口 → Task唤醒延迟
   - ADC采样完成 → 电流环计算完成延迟
