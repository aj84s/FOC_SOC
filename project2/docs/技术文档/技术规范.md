## 1. MCU选型分析

### 1.1 为什么选STM32G431

| 特性 | STM32F103RC (P1用) | STM32G431RB (P2选) | 对BLDC控制的影响 |
|------|-------------------|--------------------|-----------------|
| 内核 | Cortex-M3, 72MHz | Cortex-M4F, 170MHz | FPU加速FOC三角函数/PID计算 |
| Flash/RAM | 256KB/48KB | 128KB/32KB | RAM略小但够用，需注意内存预算 |
| 定时器 | 通用TIM1-4 | HRTIM + 通用TIM | HRTIM 184ps分辨率，FOC PWM精度极高 |
| ADC | 12-bit, 1MSPS | 12-bit, 4MSPS | 更快的电流采样，支持注入通道 |
| 运放 | 无 | 内置PGA (OPAMP) | 可省去外部电流采样运放 |
| 比较器 | 无 | 3个超快比较器 | 过流保护硬件响应 |
| CAN | 无 | FDCAN | 预留，不影响本方案 |
| 电机控制SDK | 不支持 | ST MCSDK 完全支持 | 可自动生成FOC代码框架 |

### 1.2 为什么不选其他平台
- **F103**: P1已用，且无FPU/HRTIM，做FOC太吃力
- **F407**: 有FPU但无HRTIM，电机控制不及G4系列专用
- **ESP32**: 无电机控制外设，且不是传统嵌入式MCU面试场景
- **SimpleFOC (Arduino)**: 快速原型可用，但最终代码应落在STM32上

## 2. 系统时钟配置

```
HSE: 8 MHz (NUCLEO板载晶振) / HSI16 备选
PLL: 8 MHz ÷ 1 × 85 ÷ 2 = 340 MHz (PLLCLK)
     → ÷ 2 = 170 MHz (SYSCLK)
     → ÷ 2 = 170 MHz (HCLK, AHB)
     → ÷ 2 = 85 MHz  (APB1 定时器, max 170MHz)
     → ÷ 2 = 85 MHz  (APB2 定时器, max 170MHz)

HRTIM: 170 MHz (直接来自SYSCLK的2分频, 5.88ns分辨率)
ADC: 85 MHz (APB2 ÷ 2 = 42.5MHz, 或 PLL ADC 专用时钟)
```

## 5. 任务设计

### 5.1 优先级排布

| 任务 | 优先级 | 周期/触发方式 | 理由 |
|------|--------|-------------|------|
| FOC_CurrentTask | Highest (32) | 定时器触发 (10kHz+) | 电流环必须最高频 |
| Motor_SpeedTask | AboveNormal (28) | 1ms定时 (1kHz) | 速度环，次于电流环 |
| SOC_Task | Normal (24) | 100ms定时 (10Hz) | 电池计算低频 |
| DisplayTask | BelowNormal (22) | 333ms定时 (3Hz) | 显示绝不阻塞控制 |
| CommTask | BelowNormal (20) | 100ms (10Hz) | 串口上报 |
| HealthTimer | (SW Timer) prio 2 | 100ms | 心跳+看门狗喂狗 |

### 5.2 栈空间预算

| 任务 | 分配 | 说明 |
|------|------|------|
| FOC_CurrentTask | 1024B | FOC计算+浮点运算栈深 |
| Motor_SpeedTask | 512B | PID计算 |
| SOC_Task | 512B | 库仑计数+电压校正 |
| DisplayTask | 1024B | OLED缓冲+格式化输出 |
| CommTask | 512B | 数据打包+UART发送 |
| 总RTOS栈 | ~3.5KB | 占32KB RAM的11% |

### 5.3 任务间通信

| 数据流 | 方式 | 说明 |
|--------|------|------|
| ADC完成 → FOC电流环 | 任务通知/Timer ISR | 电流采样完成触发计算 |
| FOC → 速度环 | 消息队列 (深度2) | 传递当前电流/电压 |
| 速度环 → SOC | 消息队列 (深度4) | 传递转速/电流/电压 |
| SOC → Display | 共享内存 + 互斥量 | 读取而非传递（减少拷贝） |
| SOC → Comm | 消息队列 (深度4) | 数据上报 |

## 9. 串口通信协议

### 9.1 上行帧 (MCU → PC, 100ms周期)

```
帧格式 (ASCII, 便于Python解析):
"SOC:xxx,RPM:xxxx,I:xxxx,V:xxxx,P:xxxx,T:xxx\r\n"
字段: SOC(%), RPM, 电流(mA), 电压(mV), 功率(mW), 温度(0.1°C)
示例: "SOC:85,RPM:3200,I:1250,V:11100,P:13875,T:285\r\n"
```

### 9.2 下行命令 (PC → MCU)

| 命令 | 格式 | 说明 |
|------|------|------|
| 启动电机 | `MOTOR:START,SPEED:xxxx` | 启动并设目标转速 |
| 停止电机 | `MOTOR:STOP` | 停止 |
| 查询状态 | `STATUS?` | 返回当前运行状态 |
| 设置上报周期 | `RATE:xxx` | 设置上报周期(ms) |
