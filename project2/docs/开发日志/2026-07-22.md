## 一、FOC开环速度控制

### 1. 学习与编码

- 学习灯哥FOC教程，理解控制链路：
  - Uq、角度 → 逆Park变换 → uα、uβ → 逆Clark变换 → ua、ub、uc → PWM占空比
- 编写FOC开环速度控制代码（`foc.c` / `foc.h`）

### 2. 调试记录

| Uq电压 | 目标速度 | 现象 |
|--------|----------|------|
| 2.0V | 0.1rad/s | 有力、顺时针旋转，更慢，发热严重 |
| 2.0V | 5rad/s | 有力、顺时针旋转 |
| 2.0V | 10rad/s | 有力、顺时针旋转，更快 |
| 2.0V | 12rad/s | 有力、顺时针旋转，更快 |
| 2.0V | 15rad/s | 不转，一抖一抖，蜂鸣声 |
| 3.0V | 12rad/s | 不转，一抖一抖，电源指示灯一闪一闪 |

> 没有看三相电流。

### 3. FOC调试总结

**运行效果：**
1. 开环FOC成功驱动电机，验证算法链路正确。
2. Uq=2V时运行效果最佳，5~12rad/s稳定。
3. 0.1rad/s低速运行发热明显，同步性较差。
4. Uq=2V时最高稳定速度约14rad/s。
5. 超过15rad/s后出现失步和振动现象。
6. Uq=3V时启动困难，出现电源波动。
7. Uq=1V时转矩不足，无法高速运行。
8. 速度斜坡改善启动，但无法提高极限速度。

**核心经验：**
1. 开环FOC需先验证方向和变换正确性。
2. Uq并非越大越好，存在最佳范围。
3. 低速开环容易失步并产生发热。
4. 开环最高速度受同步能力限制。
5. 速度斜坡只能改善启动过程。
6. 提高性能需引入位置反馈闭环。
7. 电压型FOC适合作为初步验证方案。

## 二、3S锂电池SOC方案讨论

### 1. 现有硬件资源梳理

**MCU**：STM32G431RBT6
- 多路12位高速ADC、3路内置运放OPAMP、多路空闲GPIO/ADC通道
- 自带内部基准电压VREFINT，适合电压采样校准

**板上已有采样电路：**
- **母线电流采样**：R19 3mΩ功率采样电阻（主回路低端），搭配OPAMP1运放，可测充放电电流
- **母线电压检测**：R30 75kΩ + R33 10kΩ分压网络，中点接ADC，3S满电12.6V → 分压后约1.48V，在3.3V量程内
- **温度检测**：TEMP（测驱动板功率管温度，非电芯温度）
- **I2C接口**：已引出I2C1_SCL / I2C1_SDA引脚

**板上短板：**
- 只采样总母线电压，无3路独立分压网络分别采集3串电芯单体电压
- 无电池均衡硬件
- 无独立电池NTC测温

### 2. 关键Q&A

**Q1：PCB能改吗？**
- 不能改，是买的成品电控+驱动一体板。所有方案需基于现有硬件+外挂模块。

**Q2：母线电流 = 电池放电总电流吗？**
- 是同一回事。R19 3mΩ是母线低端采样电阻，串联在三相全桥下桥臂公共端和电源负极之间。电池正极流出的所有电流，经过三相桥驱动电机，最终都会汇总流过这个电阻回到电池负极。
- R19上的电流 = 三相桥总直流电流 = 电池放电总电流（非U/V/W单相交流相电流）。

**Q3："母线分压ADC读3S总电压"是哪个模块？**
- 对应原理图电压检测部分：R30 75kΩ + R33 10kΩ串联在VBUS和GND之间，分压中点接ADC。3S满电12.6V → 分压后≈1.48V，在ADC安全范围内。代码读取ADC值后按比例反算即可得到电池总电压。

**Q4：不带载对SOC有什么影响？**
- 是利好。空载下放电电流很小，电池极化效应极弱，端电压接近真实OCV。只用电压查表做SOC，误差比大电流带载小很多，不会出现严重电量跳变。

**Q5：电流积分说的是什么电流？现有电路能测吗？加INA226行不行？**
- 指的是电池直流总放电电流（母线电流）。
- **现有电路**：R19 3mΩ + OPAMP1可以测，但不适合SOC，原因：
  - 原为电机控制设计（单电阻采样、过流保护），量程偏向大电流，毫安级小电流精度差
  - 采样值与PWM开关强相关，纹波大，需复杂同步采样
  - 代码与电机控制深度耦合，修改易影响驱动功能
- **INA226方案**：非常合适。独立I2C接口的电流/电压/功率采集芯片，串联在电池正极总线上：
  - 完全独立工作，不干扰现有电机驱动
  - 同时测量电池总电压和放电电流，小电流精度远高于板上电机专用采样
  - 直接I2C读数，无需处理ADC校准、运放偏移
  - 板上已引出I2C1_SCL/I2C1_SDA，直接接线，不用改PCB

### 3. 两种落地方案

#### 方案A：极简版（纯电压查表，无需额外硬件）
- 读取板上现有母线分压ADC，计算3S电池总电压
- 代码预存「3S三元锂总电压-SOC对照表」
- 空载状态下查表输出电量百分比
- **优点**：零额外硬件，直接写代码
- **缺点**：精度一般，仅作参考

#### 方案B：推荐版（INA226 + 电压 + 库仑积分）
- **硬件**：INA226串联在电池供电总线上，SDA/SCL接I2C1，供电接3.3V
- **算法**：
  1. 开机/长时间静置时：INA226读电池电压，OCV查表校准初始SOC
  2. 运行时：INA226读实时电流，定时库仑积分更新SOC（`ΔQ = I × Δt`）
  3. 定期静置时自动用电压校准，消除积分漂移
- **精度**：±5%以内，远优于纯电压方案
- **不影响电机驱动代码**

### 4. 结论

选择**方案B（INA226 + 电压 + 库仑积分）**，原因：
- 手头已有INA226模块
- 不改PCB，I2C直连
- 精度显著优于纯电压方案
- 独立工作，不干扰电机驱动代码
