## Phase 1: 电机基本控制 (预计 2周, 07-17 → 07-31)

**目标:** BLDC电机能稳定旋转，实现速度闭环控制

### Step 1.1 — PWM输出验证 (1天)
- [ ] `MotorControl/bldc_commutation.c/h` — 创建模块骨架
- [ ] 定时器初始化完成后，先用示波器/逻辑分析仪验证6路PWM:
  - 频率 20kHz
  - 死区时间 500ns (上下桥不会同时导通)
  - 占空比可调 (0-95%)
- [ ] 编写 `BLDC_SetDutyCycle()` 函数
- **验证:** 逻辑分析仪捕获6路PWM波形正确

### Step 1.2 — 6步换相开环控制 (3天)
- [ ] 确定电机相序: 用万用表确认三相绕组，标记A/B/C
- [ ] 6步换相状态机: 
  ```
  A+B- → A+C- → B+C- → B+A- → C+A- → C+B- → A+B- → ...
  ```
- [ ] 换相表: 6步 × 6路PWM的开关状态
- [ ] 定时器中断触发换相 (频率=转速/60×极对数, 或固定换相周期)
- [ ] `BLDC_CommutationStep()` — 每次中断执行一步
- [ ] 开环运行: 固定换相频率，电机空载旋转
- **验证:** 电机平稳旋转，无抖动/堵转，电流 < 1A

### Step 1.3 — ADC电流采样 (2天)
- [ ] 配置ADC1 + DMA (PA0, 母线电流)
- [ ] 双缓冲: bufA[256] / bufB[256]
- [ ] 同步采样: PWM中心对齐触发ADC
- [ ] `BLDC_ReadPhaseCurrent()` — 返回A/B/C相电流
- **验证:** 串口输出电流值，空载/负载时能看到电流变化

### Step 1.4 — 转速测量 (1天)
- [ ] 基于换相频率计算转速: RPM = (换相频率/6) × 60 / 极对数
- [ ] 或无传感器反电动势检测 (备用方案)
- [ ] `BLDC_GetSpeed()` — 返回当前 RPM
- **验证:** 串口输出RPM，手动改变PWM占空比，RPM相应变化

### Step 1.5 — PI速度闭环 (3天)
- [ ] `BLDC_PISpeedControl(targetRPM)` — PI控制器
- [ ] P项: 误差比例 → 调节响应速度
- [ ] I项: 误差积分 → 消除稳态误差，积分限幅防饱和
- [ ] PI输出 → 限制 → 更新PWM占空比
- [ ] PI参数整定: 先P后I，观察阶跃响应
- **验证:** 设定目标转速 → 电机在1秒内到达目标RPM ±5%

### Step 1.6 — FOC升级（可选，视进度） (2天)
- [ ] 如果6步换相+速度环已稳定，开始升级FOC
- [ ] 先用 SimpleFOC 库快速跑通，验证硬件没问题
- [ ] Clarke/Park变换 → PI电流环 → SVPWM
- [ ] 从SimpleFOC逐步迁移到自写代码
- **验证:** FOC模式下电机运行更安静、扭矩更平滑

**Phase 1 验证门禁:**
- [ ] 电机在任意设定转速 (500-5000RPM) 稳定运行
- [ ] 速度环阶跃响应 < 1秒
- [ ] 空载电流 < 0.5A
- [ ] 运行10分钟无过流/过热/死机

## Phase 2: 电池监测与SOC估算 (预计 2周, 08-01 → 08-14)

**目标:** 电池数据采集正常，SOC能跟随放电过程变化

### Step 2.1 — INA226 I2C驱动 (2天)
- [ ] `BatteryMonitor/ina226.c/h` — 创建模块
- [ ] I2C1 初始化 (PB6/PB7, 400kHz)
- [ ] INA226 初始化: 配置寄存器 (平均值/转换时间/模式)
- [ ] INA226 校准: 基于采样电阻0.1Ω计算Calibration值
- [ ] `INA226_ReadBusVoltage()` — 读取总线电压
- [ ] `INA226_ReadCurrent()` — 读取电流 (分流电压/采样电阻)
- [ ] `INA226_ReadPower()` — 读取功率
- **验证:** 串口输出电压/电流/功率值，与万用表实测对比误差 < 1%

### Step 2.2 — 库仑计数 (2天)
- [ ] `BatteryMonitor/soc.c/h` — 创建模块
- [ ] `SOC_Init(2200mAh)` — 初始化，设定电池容量
- [ ] 100ms 周期: `SOC_Update()` → 读取电流 → 积分累加
- [ ] 库仑计数: `accumulated_mAh += current_mA × (100ms / 3600s)`
- [ ] `SOC_GetPercentage()` — 返回 SOC = (2200 - accumulated) / 2200 × 100
- [ ] 充放电方向: 正电流=放电, 负电流=充电
- **验证:** 电池放电过程 SOC 从100%逐渐下降到90%、80%...（需实际放电测试）

### Step 2.3 — 电压法OCV校正 (2天)
- [ ] 开路电压检测: 电流 < 50mA 持续 5秒 → 判定为近似开路
- [ ] OCV-SOC查表 (3S LiPo典型曲线):
  ```
  12.60V → 100%   11.70V → 60%    10.80V → 20%
  12.30V → 90%    11.40V → 50%    10.50V → 10%
  12.00V → 80%    11.10V → 40%    9.00V  → 0%
  11.85V → 70%    10.95V → 30%
  ```
- [ ] 满足开路条件时: 用OCV查表值校正库仑计数SOC
- [ ] 加权融合: `SOC = α×SOC_coulomb + (1-α)×SOC_ocv` (α 根据电流大小动态调整)
- **验证:** 放电到某SOC → 静置 → OCV校正生效 → SOC值修正

### Step 2.4 — OLED显示 (2天)
- [ ] `Display/oled_display.c/h` — 创建模块（复用P1 SSD1306驱动）
- [ ] 页0: 实时数据 (SOC% / RPM / 电流A / 电压V)
- [ ] 页1: 电池详情 (累计Ah / 瞬时功率W / 电压跌落)
- [ ] 页2: 系统状态 (运行时间 / 温度 / 错误计数)
- [ ] 3秒自动翻页 + PB0按键手动翻页
- **验证:** OLED三页切换正常，数据刷新正常

**Phase 2 验证门禁:**
- [ ] INA226 数据正确 (与万用表对比)
- [ ] SOC 跟随放电过程持续下降
- [ ] OCV 校正后 SOC 误差缩小
- [ ] OLED 三页显示正常

## Phase 3: 融合算法与完善 (预计 2周, 08-15 → 08-25)

**目标:** 电机-SOC融合补偿生效，系统完整可演示

### Step 3.1 — 电机参数-SOC关联建模 (3天)
- [ ] 实验: 不同SOC下(100%/75%/50%/25%)，记录同一油门PWM下的稳态转速
- [ ] 建立查找表: `油门×SOC → 预期转速`
- [ ] 实际转速与预期转速的偏差 → 用于校正SOC
- [ ] `SOC_CompensateWithMotor(motorRPM, throttlePercent)`:
  ```
  如果同样油门, 转速比预期低20% → 电池可能比OCV显示的更空
  如果同样油门, 转速比预期高10% → 电池可能刚充满（电压虚高）
  ```
- **验证:** 标定实验完成，补偿模型能在测试集上改善SOC精度
