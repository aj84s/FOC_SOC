## 📖 项目简介

电动自行车已成为中国主流交通工具，但大多数厂商的电量显示**严重不准**——明明显示4格电，骑了一周还是4格，却随时可能断电。

**核心洞察**：同样油门开度下，不同电量时电机转速明显不同。电机转速-电流-电池SOC之间存在可建模的耦合关系。

本项目构建了一个**完整的嵌入式闭环系统**：BLDC电机模拟轮毂电机，INA226实时监测电池，利用库仑计数+OCV开路电压+**电机参数补偿**三重融合算法改善SOC估算精度，OLED本地显示+串口数据上报。

**面试目标岗位**：嵌入式软件工程师 | 重点展示：控制闭环设计 / 真实问题出发的系统思维 / 多源数据融合

## ✨ 核心特性

### 🔧 硬件能力
| 模块 | 方案 | 亮点 |
|------|------|------|
| **MCU** | STM32G431RB (Cortex-M4F, 170MHz) | HRTIM + 内置PGA/比较器，电机控制专用 |
| **电机控制** | BLDC 6步换相 → FOC升级 | DRV8313/MS8313 三半桥驱动，硬件过流保护 |
| **电池监测** | INA226 I2C（16位, 0.1Ω采样电阻） | 数字输出免模拟调理，同步测电压+电流+功率 |
| **本地显示** | SSD1306 OLED 128×64 I2C | 三页切换：实时数据/电池详情/系统状态 |
| **数据上报** | USART2 115200bps | ASCII格式，Python上位机实时绘图 |

### 🧠 软件亮点
- **FOC磁场定向控制**：Clarke/Park变换 → PI电流环 → SVPWM → 速度闭环
- **SOC三重融合算法**：库仑计数（主） + OCV校正（辅） + **电机参数补偿（创新点）**
- **FreeRTOS多任务架构**：5个优先级任务 + 软件定时器 + 消息队列/互斥量通信
- **系统可靠性**：IWDG看门狗 / 过流欠压保护 / 故障异常诊断 / 任务栈水位监控

## 🚀 快速开始

- **IDE**: Keil MDK-ARM 5.38+
- **工具链**: ARM Compiler 6
- **硬件**: NUCLEO-G431RB 开发板
- **调试器**: ST-Link/V2（NUCLEO板载）

## 🎯 技术面试要点

| 维度 | 面试官可能问 | 你的回答方向 |
|------|------------|------------|
| **为什么选G431** | 为什么不用F103/F407/ESP32? | HRTIM+内置PGA做FOC更合适，Cortex-M4F带FPU加速PID，对比分析过四款芯片 |
| **FOC理解** | 6步换相和FOC的区别？ | 6步是方波驱动扭矩脉动大，FOC是正弦波驱动更平滑高效，Clarke/Park变换原理 |
| **SOC算法** | 怎么比纯电压法更准？ | 库仑计数为基础+OCV校正累计误差+电机转速补偿负载效应，三重融合互补 |
| **RTOS设计** | 为什么用FreeRTOS？任务优先级怎么排？ | 5个任务按实时性分层：电流环最高频→速度环→SOC→显示→通信，时间隔离保证安全 |
| **系统可靠性** | 怎么处理异常？ | IWDG+过流硬件保护+欠压自动停机+故障异常诊断输出，多层防护 |
| **工程素养** | 你的开发流程？ | 需求文档→技术规范→设计规范→开发路线图→分Phase→每Phase有验证门禁→每日日志 |
